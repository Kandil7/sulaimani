## 1. نظرة عامة على المنتج

### 1.1 ملخص تنفيذي
نظام إدارة متكامل لـ **صيدلية السليماني** — محل متخصص في بيع الأدوية البشرية والمبيدات الزراعية. التطبيق يعمل على **Windows Desktop** ويهدف إلى استبدال الإدارة اليدوية بنظام رقمي يضمن دقة المخزون، تتبع الصلاحيات، وإصدار الفواتير بشكل فوري. [chisellabs](https://chisellabs.com/blog/product-requirement-document-prd-templates/)

### 1.2 المشكلة المُحلَّة
| المشكلة الحالية | الحل في التطبيق |
|----------------|----------------|
| تتبع المخزون يدوياً بالورق | قاعدة بيانات محلية محدّثة real-time |
| اكتشاف المنتجات المنتهية متأخراً | تنبيهات تلقائية قبل 30 يوم |
| الفواتير يدوية أو غير موجودة | إصدار PDF فوري وشاركه على واتساب |
| متابعة ديون العملاء صعبة | سجل عملاء مع رصيد الدين |
| لا توجد تقارير مبيعات | تقارير يومية/شهرية مرئية |

### 1.3 الأهداف القابلة للقياس
- تقليل الوقت اللازم لإتمام عملية بيع من 5 دقائق → **أقل من 60 ثانية**
- صفر منتجات تُباع بعد انتهاء صلاحيتها
- توفير تقرير يومي دقيق بدون جهد يدوي
- استرداد رصيد أي عميل في **أقل من 5 ثواني**

## 5. المتطلبات غير الوظيفية (Non-Functional Requirements)

### 5.1 الأداء
| المتطلب | المعيار |
|--------|--------|
| سرعة البحث في المنتجات | < 200ms |
| تحميل شاشة POS | < 500ms |
| إتمام عملية البيع وتوليد الفاتورة | < 2 ثانية |
| تحميل التقارير الشهرية | < 3 ثواني |
| سعة قاعدة البيانات | +10,000 منتج، +50,000 فاتورة |

### 5.2 الموثوقية
- العمل **100% Offline** بدون إنترنت
- لا يُفقد أي بيانات عند الإغلاق المفاجئ
- **Auto-save** بعد كل عملية

### 5.3 قابلية الاستخدام (Usability)
- واجهة عربية كاملة **RTL**
- إتمام عملية بيع كاملة بـ **3 خطوات فقط**
- رسائل خطأ واضحة بالعربي
- Keyboard shortcuts للعمليات المتكررة

### 5.4 الأمان
- قاعدة البيانات محلية مشفرة (Isar encryption)
- تأكيد قبل حذف أي بيانات
- لا يُمكن حذف منتج له فواتير مرتبطة (Phase 2)

### 5.5 قابلية التوسع
- Clean Architecture يسمح بإضافة Cloud Sync لاحقاً
- Repository Pattern يسمح باستبدال قاعدة البيانات
- Feature-based folder structure لإضافة features جديدة بسهولة [chisellabs](https://chisellabs.com/blog/product-requirement-document-prd-templates/)

### 5.6 التوافق
| البيئة | المتطلب |
|--------|--------|
| النظام | Windows 10 / 11 |
| الدقة | 1280×720 كحد أدنى، مُحسَّن لـ 1920×1080 |
| الذاكرة | 4GB RAM كحد أدنى |
| التخزين | 500MB للتطبيق + مساحة للبيانات |

## 7. User Stories — القصص الوظيفية

### Epic 1: إدارة المنتجات
```
US-01: كصاحب محل، أريد إضافة منتج جديد بكل بياناته
       حتى يظهر فوراً في قائمة المنتجات وشاشة البيع
       
       Acceptance Criteria:
       ✅ الفورم يتحقق من الحقول المطلوبة
       ✅ المنتج يظهر في القائمة بدون reload
       ✅ رسالة نجاح تظهر بعد الحفظ

US-02: كصاحب محل، أريد تعديل سعر البيع لمنتج موجود
       بدون فقدان باقي البيانات
       
       Acceptance Criteria:
       ✅ Dialog يُحمَّل ببيانات المنتج الحالية
       ✅ بعد الحفظ، السعر الجديد يظهر فوراً في الجدول
```

### Epic 2: عملية البيع
```
US-03: كصاحب محل، أريد إتمام عملية بيع بأقل عدد خطوات
       حتى أخدم العميل بسرعة
       
       Acceptance Criteria:
       ✅ البحث يبدأ عند أول حرف
       ✅ نقرة واحدة تضيف للسلة
       ✅ F2 يفتح نافذة الدفع
       ✅ الفاتورة تتولد خلال ثانيتين

US-04: كصاحب محل، أريد تحذيراً عند محاولة بيع منتج منتهي
       حتى لا أبيع منتجاً غير صالح للاستخدام
       
       Acceptance Criteria:
       ✅ تحذير واضح يظهر باللون الأحمر
       ✅ يمكن تجاهل التحذير والمتابعة
       ✅ يُسجَّل في الـ Log
```

### Epic 3: التنبيهات
```
US-05: كصاحب محل، أريد رؤية كل المنتجات قريبة الانتهاء
       حتى أتواصل مع المورد في الوقت المناسب
       
       Acceptance Criteria:
       ✅ القائمة مرتبة بالأقرب انتهاءً أولاً
       ✅ Badge في الـ Sidebar يعكس العدد الحقيقي
       ✅ اللون يتغير: أصفر (30 يوم)، أحمر (منتهي)
```
